**4）facebook 开源的源码控制工具 [Sapling](https://github.com/facebook/sapling "开源项目 Snapling")**

它和 git 存在如下差别：

- Sapling 不需要或鼓励使用命名的本地分支；
- 没有 staging area；
- clone/push 时，不会下载所有的仓库数据；
- 对回退撤销的支持更好（类似 git 的 checkout/reset/reflog）；
- 不使用 rebase；
- Sapling 通常每个命令做一件事；
- Sapling 跟踪了更多 commit 历史等等。
